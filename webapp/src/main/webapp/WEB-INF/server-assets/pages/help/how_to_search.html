1. Search for experiments

Search by accession or keyword

Enter an experiment accession number (e.g. E-MEXP-568) or keyword (e.g. RNAi, "breast cancer", "p53 knockout") in the query box at the top of the ArrayExpress Experiments page.

  • Put quotes around multiple keywords if you want to find experiments where these words are found next to each other e.g. "breast cancer".
  • Entering multiple words without quotes will retrieve experiments where both keywords are found but they are not necessarily adjacent e.g. mouse leukemia.

Filtering results

Experiments can be filtered using the drop down menus at the top of the results table. They can be filtered by

  • organism
  • array design used
  • molecule (DNA, RNA, amplicon, metabolite, protein)
  • technology (array, high-throughput sequencing, mass spectrometry)
  • ArrayExpress data only - experiments submitted directly to ArrayExpress, not imported from the NCBI Gene Expression Omnibus (GEO). Use the 'ArrayExpress data only' checkbox to activate this filter. For more information about the data we import from GEO see the GEO data help page.

After selecting a filter, click on the 'Filter' button on the right hand side to filter experiments. To remove a filter, re-select the top option from the list and then click on the 'Filter' button again to requery ArrayExpress.

2. Search for array designs

Array designs, also known as platform designs, can be searched on the Arrays page

You can search array designs using the following:

  • ArrayExpress accession: It has a format of A-xxxx-n, where xxxx represents a four letter code and n is a number e.g. "A-MEXP-1234", "A-AFFY-44".
  • Organism: The scientific (Latin) name of an organism, e.g. "Homo sapiens", "Saccharomyces cerevisiae" represented on the array design.
  • Array name: For commerically-available arrays, this will be the official name provided by the manufacturer, e.g. Agilent "SurePrint G3 Rat CGH Microarray". Parts of names can also be searched, e.g. "Rat SurePrint" if you are not sure of the precise name.
  • Array description: Examples are the array's catalogue number, Design ID/Grid name (for Agilent arrays), the purpose of the array (e.g. "CGH", "expression").
  • Array manufacturer name: Examples are "Affymetrix", "Agilent", "NimbleGen" and "Illumina". This can also be the name of the laboratory which printed the microarray.
  • Array provider name: The name or email address of the person who submitted the array design to ArrayExpress.

ArrayExpress contains array designs submitted directly to us and those imported from NCBI Gene Expression Omnibus (see the GEO data help page for more details). You can toggle the check box "ArrayExpress data only" to search for direct submission only or all array designs.

3. Search for private experiments

Data can be kept private in ArrayExpress until an associated paper is published. After an experiment or custom array design is loaded into ArrayExpress the submitter is sent details of login accounts for themselves, and for journal editors and reviewers so that they can view the data before it is publicly available.

To login click on the "Login" link at the right hand side of the top menu bar.

A pop up box will appear for you to enter the username and password you have been sent.

If the 'Remember me' box is checked, the login details will be remembered by your web browser, even if you close the browser and shut down your computer.

Once you've logged in, search for the experiment you would like to view by entering its accession number in the search box and click "Search"

 

Forgotten passwords and reviewers with no password

If you have forgotten your ArrayExpress data access account login and/or password, please email the curation team at miamexpress@ebi.ac.uk with the accession number of the experiment or array design you wish to view.

If you are a reviewer and have not been provided with an ArrayExpress login to access private data connected to a publication, please contact the data submitter, via the journal, to request this information. We cannot provide access to private data without first getting authorization from the submitter or journal.

When do we make data public?

Data is made public when the submitter gives us permission to do so, or if we find that the data has been referenced in a published article. Please also read our data access policy for more information.
